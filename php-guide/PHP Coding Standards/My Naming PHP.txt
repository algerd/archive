1. Объявляйте переменные рядом с местом их первого использования
2. Если метод возвращает какие-то важные значения, напр. свойства, то он должен именоваться getName() (Name - имя свойства)
3. Если метод задаёт какие-то важные значения, напр. свойств, то он должен именоваться setName (Name - имя свойства)
4. Eсли метод возвращает булево значение как результате выполнения условий, то он должен именоваться isIfname (Ifname - название условия)
5. Если метод возвращает булево значение как результат проверки на наличие элементв, то он должен именоваться hasName (Name - название проверяемого элементв)
6. Глаголы именования методов (функций):
    - add или insert - добавить
    - join - объединить
    - delete или remove - удалить
    - create - создать

7. Все методы и функции именуются верблюжьей нотацией:
    - первое слово с маленькой буквы - глагол (напр. add) или предлог (напр. to)
    - второе и последующие слова с большой буквы - существительное

8. Интерфейс группирует классы по набору функционала (методов), поэтому мнтерфейсы называют по
    слову, описывающему этот функционал, с окончанием able (способный). Например, интерфейс группирует классы по 
    методу  масштабирования фигуры (scale()), т.е. все классы от этого интерфейса будут способны масштабировать или
    по английски они будут Scaleable. И таким образом название интерфейса будет - Scaleable. 
    Никаких приставок и окончаний типа I, Inter или Interface не использовать при именовании интерфейса!
