			Функции расширения SPL - http://www.php.net/manual/ru/ref.spl.php


spl_object_hash — Возвращает хеш-идентификатор для объекта. При смене сессии он обновляется, 
    т.е. он привязан к сессии (в отличие от md5(), которая не зависит от сессий).
    Этот id можно использовать в качестве хэш-ключа для хранения объектов или для идентификации объектов.

Информационные функции (дополнение к существующим функциям Class Object Information)
------------------------------------------------------------------------------------
class_implements — Возвращает список интерфейсов, реализованных в заданном классе
class_parents — Возвращает список родительских классов заданного класса	
class_uses — Возвращает список трэйтов, используемых заданным классом
spl_classes  - 	Возвращает массив c доступными в настоящее время классами SPL.


Функции для работы с итераторами:
---------------------------------
iterator_apply - вызывает функцию для каждого элемента в итераторе. Позволяет аналогично foreach обходить итератор,
    передавая кажый элемент итератора в callback функцию. Более гибко и изящно заменяет стандартный foreach.
    !!! Очень полезная функция !!!

iterator_count - Подсчитывает количество элементов в итераторе. Аналогичен count() для массива.

iterator_to_array — Копирует итератор в массив		
    Копирует элементы итератора в массив. Фактически это обратное преобразование итератора в массив.
				

Функции	автозагрузки:
---------------------			
spl_autoload — Реализация метода __autoload() по умолчанию - встроенный автозагрузчик.
spl_autoload_extensions — Регистрация и вывод расширений файлов для spl_autoload()	
spl_autoload_register — Регистрирует заданную функцию в качестве реализации метода __autoload()
spl_autoload_unregister — Отмена регистрации функции в качестве реализации метода __autoload()
spl_autoload_call — Попытка загрузить описание класса всеми зарегистрированными методами __autoload()
spl_autoload_functions — Получение списка всех зарегистрированных фунций __autoload()

Для формирования автозагрузки используютя:
spl_autoload_extensions - для указания расширения
spl_autoload_register - для регистрации функций и методов автозагрузки
spl_autoload - для формирования автозагрузки из include_path
