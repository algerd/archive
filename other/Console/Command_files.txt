
Командный файлы имеют расширение [namefile].cmd
Запускаютя в консоли

с:\>[путь][namefile].cmd   ввоод

Консоль выполнит последовательно все команды командного файла.
В команном файле записsвается список команд консоли:
	echo off
	echo "создание директории"
	mkdir main\test
	echo on
 
Подробное описание команда смотреть с помощью help команда  
 
Помещение комментариев в пакетные файлы и файл CONFIG.SYS.
	REM [текст] 
 

Вывод сообщений и переключение режима отображения команд на экране:
	ECHO [ON | OFF] off - не будут отображаться команды в консоли, кроме сообщений в echo "[]"
	ECHO [сообщение]

Значок @ перед командой не будет выводить в консоль данную команду, но будет её исполнять
	@mkdir main\test
	
Оператор условного выполнения команд в пакетном файле.
	IF [NOT] ERRORLEVEL число команда
	IF [NOT] строка1==строка2 команда
	IF [NOT] EXIST имя_файла команда
	
		NOT             Указывает, что Windows должна выполнитьэту команду, только если условие является ложным.
		ERRORLEVEL число  Условие является истинным, если код возврата последней выполненной программы не меньше указанного числа.
		строка1==строка2  Условие является истинным, если указанные строки совпадают.
		EXIST имя_файла   Условие является истинным, если файл с указанным именем существует.
		
	Предложение ELSE должно располагаться в той же строке, что и команда, следующаяза ключевым словом IF. Например:
			IF EXIST имя_файла. (
				del имя_файла.
			) ELSE (
				echo имя_файла. missing.
			)
			
Выполнение указанной команды для каждого файла набора.
	FOR %переменная IN (набор) DO команда [параметры]

		%переменная Однобуквенный подставляемый параметр.
		(набор)      Определяет набор, состоящий из одного или нескольких файлов.  Допускается использование подстановочных знаков.
		команда      Команда, которую следует выполнить для каждого файла.
		параметры    Параметры и ключи для указанной команды.

	В пакетных файлах для команды FOR используется запись%%переменная вместо %переменная. Имена переменных учитывают регистр букв (%i отличается от %I).
	
Использование пакетного (командного) файла как функции:
	с:\>[путь][namefile].cmd [arg1] [arg2] ...[arg9]
	Пути с пробелами надо обрамлять ковычками

	с:\>funсtion_copy.cmd "/test 2" txt   - передаётся в качестве аргументов %1 = /test и %2 = txt в пакетный файл. 
	Аргументов может быть 9 и им будут соостветствовать переменные в пакетном файле с именами %1 - %9
